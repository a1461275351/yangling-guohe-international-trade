## 二、P0 阶段：安全与可运行（立即修复）

### P0-1 biz_template 表缺少 4 个字段
- **现状**: Template 实体有 filePath/fileName/fileSize/pdfPath，但 SQL 表无此列
- **影响**: 模板上传/下载/预览接口运行时报 Unknown column
- **修复**: 在 init.sql 中为 biz_template 添加 file_path/file_name/file_size/pdf_path 列
- **状态**: ✅ 已修复

### P0-2 JWT 过期时间计算错误（~1000天）
- **现状**: application.yml 配置 86400000（毫秒），JwtUtil.java 又乘以 1000
- **影响**: Token 有效期约 1000 天，泄露后风险窗口极大
- **修复**: 将 expiration 配置改为秒（86400），保留代码中 *1000 转毫秒
- **文件**: `backend/src/main/resources/application.yml:33`, `JwtUtil.java:22`
- **状态**: ✅ 已修复

### P0-3 密码重置硬编码 "123456"
- **现状**: UserService.java 两处重置密码固定为 "123456"
- **影响**: 不安全，不符合国企系统安全基线
- **修复**: 改为生成 12 位随机密码（SecureRandom），返回给操作者
- **文件**: `UserService.java`
- **状态**: ✅ 已修复

### P0-4 注册接口允许提交任意角色（越权风险）
- **现状**: RegisterDTO.role 无校验，前端可提交 ADMIN
- **影响**: 审批误操作可能创建管理员账号
- **修复**: 后端强制限制注册只能申请 ENTERPRISE 角色
- **文件**: `UserService.java`
- **状态**: ✅ 已修复

### P0-5 全局异常处理泄露内部信息
- **现状**: GlobalExceptionHandler 返回 `"系统内部错误: " + e.getMessage()`
- **影响**: 泄露 SQL 结构、类名、路径
- **修复**: 系统异常只返回 "系统繁忙，请稍后再试"
- **文件**: `GlobalExceptionHandler.java`
- **状态**: ✅ 已修复

### P0-6 报关单控制器无权限注解
- **现状**: CustomsController 所有方法（approve/reject/release 等）均无 @RequireRole
- **影响**: 任何登录用户可审批/驳回/放行报关单
- **修复**: approve/reject/release 限制为 ADMIN/GUOHE
- **文件**: `CustomsController.java`
- **状态**: ✅ 已修复

### P0-7 /uploads/** 静态资源无鉴权
- **现状**: WebMvcConfig 直接映射 /uploads/** 为静态目录
- **影响**: 合同、报关单等敏感文件可通过 URL 猜测访问
- **修复**: 移除静态映射，文件通过 FileController 鉴权下载
- **文件**: `WebMvcConfig.java`
- **状态**: ✅ 已修复

### P0-8 OnlyOffice 回调未认证 + SSRF 风险
- **现状**: callback 路径排除 JWT 检查，OnlyOffice JWT secret 为空
- **影响**: 可伪造回调覆盖模板文件，SSRF 风险
- **修复**: 添加回调 JWT/Authorization 验证 + SSRF 地址过滤
- **文件**: `OnlyOfficeController.java`
- **状态**: ✅ 已修复

### P0-9 CORS 允许任意来源
- **现状**: `allowedOriginPatterns("*")` + `allowCredentials(true)`
- **影响**: 生产环境跨域攻击面过大
- **修复**: 通过 `cors.allowed-origins` 配置项限制域名
- **文件**: `CorsConfig.java`, `application-prod.yml`
- **状态**: ✅ 已修复

### P0-10 业务异常返回 500 错误码
- **现状**: BusinessException 被 GlobalExceptionHandler 捕获返回 500
- **影响**: 密码错误、数据不存在等被误判为系统故障
- **修复**: BusinessException 默认返回 400
- **文件**: `BusinessException.java`
- **状态**: ✅ 已修复

## 三、P1 阶段：权限与前端完善

### P1-1 前端路由守卫不校验角色
- **现状**: router beforeEach 只检查 token，meta.roles 完全未使用
- **影响**: 任何登录用户可通过 URL 访问管理页面
- **修复**: beforeEach 中增加角色校验，无权限跳转 /403 页面
- **文件**: `frontend/src/router/index.js`
- **状态**: ✅ 已修复

### P1-2 前端侧边栏角色过滤不一致
- **现状**: sidebar 用 `v-if="isAdmin"` 隐藏管理菜单，但路由允许 GUOHE
- **影响**: GUOHE 用户看不到菜单但可通过 URL 访问
- **修复**: sidebar 改为 `isAdmin || isGuohe` 与路由 roles 保持一致
- **文件**: `frontend/src/layout/AppLayout.vue`
- **状态**: ✅ 已修复

### P1-3 后端权限注解与前端路由不一致
- **现状**: UserController/TenantController/UserApplyController 只允许 ADMIN
- **影响**: GUOHE 用户前端能看到页面但后端接口被拒
- **修复**: 权限注解统一为 ADMIN + GUOHE
- **文件**: `UserController.java`, `TenantController.java`, `UserApplyController.java`
- **状态**: ✅ 已修复

### P1-4 文件上传缺少类型白名单
- **现状**: FileService 只检查大小，不校验文件类型
- **影响**: 可上传任意类型文件，安全风险
- **修复**: 添加文件扩展名白名单（pdf/doc/xls/jpg/png等）+ 空文件和文件名校验
- **文件**: `FileService.java`
- **状态**: ✅ 已修复

### P1-5 基础配置数据缺失
- **现状**: init.sql 无 COUNTRY/CURRENCY/TRANSPORT_MODE 等基础字典
- **影响**: 前端下拉框无数据，合同/订单/报关流程缺少基础字典
- **修复**: 在 init.sql 中添加 7 类共 70+ 条基础配置种子数据
- **文件**: `sql/init.sql`
- **状态**: ✅ 已修复

### P1-6 前端 API 拦截器缺少 403 处理
- **现状**: request.js 只处理 401，403 无特殊处理
- **修复**: 响应拦截器和错误处理器均添加 403 处理，提示"无权限执行此操作"
- **文件**: `frontend/src/utils/request.js`
- **状态**: ✅ 已修复

### P1-7 CustomsDeclaration 实体缺少 updateTime 字段
- **现状**: SQL 有 update_time，实体未声明
- **修复**: 添加 updateTime 字段
- **文件**: `CustomsDeclaration.java`
- **状态**: ✅ 已修复

### P1-8 前端 dashboard 吞掉 API 错误
- **现状**: dashboard onMounted 中 catch(e) {} 空捕获
- **修复**: 改为 console.warn 输出错误信息
- **文件**: `frontend/src/views/dashboard/index.vue`
- **状态**: ✅ 已修复
