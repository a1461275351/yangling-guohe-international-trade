四、PVE 测试部署建议
1. 推荐架构
对公司基础⽐较薄弱的情况，建议先⾛最稳妥的⽅案：
PVE 物理服务器
  └── Ubuntu Server 测试虚拟机
        ├── Docker / Docker Compose
        ├── MySQL 8.0
        ├── Java Spring Boot 后端
        ├── Nginx 前端静态⽂件
        └── uploads ⽂件⽬录
不要⼀开始就上复杂的  Kubernetes 。当前团队更需要 “ 能部署、能备
份、能恢复、能看⽇志、能让⼈⽤起来 ” 。
2. 测试环境推荐配置
使⽤⼈数 CPU 内存 磁盘 说明
5-10 ⼈内部试
⽤4 核 8GB 100-200GB
SSD可以⽤于第⼀轮
流程验证
20-50 ⼈测试 8 核 16GB 300-500GB
SSD推荐给业务、财
务、单证共同测
试
后续⽣产起步 8-16 核 32GB 1TB SSD 起 需要加备份盘和
监控
测试环境的磁盘建议单独规划：
系统盘：80-100GB

---

数据盘：MySQL 数据、上传⽂件、⽇志，建议  200GB 起
备份盘：每⽇数据库备份、上传⽂件备份，建议独⽴挂载
3. 测试部署最低条件
上 PVE 测试前，⾄少满⾜：
PVE 上建好⼀台  Ubuntu Server 虚拟机。
虚拟机能访问内⽹，团队电脑能访问该虚拟机  IP 。
Docker 和  Docker Compose 可⽤。
MySQL 数据⽬录、上传⽬录、⽇志⽬录做持久化。
后端和前端使⽤最新构建产物，不要直接使⽤旧的
deploy/dist。
修改默认管理员密码。
修改 JWT 密钥。
明确测试数据和正式数据分开。
每天⾄少备份⼀次数据库。
