# 外综服平台 v1.0.4 升级部署指南

> 适用场景：从 1.0.0 升级到 1.0.4
> 目标服务器：192.168.3.65
> 预计停机时间：3-5 分钟

---

## 一、升级包清单

| 文件 | 大小 | 说明 |
|------|------|------|
| `trade-platform-1.0.4.jar` | 71.9 MB | 后端 jar（Java 17 编译，含 P3 全部模块） |
| `dist/` | 87 个文件 | 前端打包产物 |
| `application-prod.yml` | 模板 | 生产环境配置（**保留服务器原有的，不要覆盖**） |
| `nginx.conf` | - | nginx 配置参考 |
| `sql/init.sql` | - | 完整数据库初始化脚本（含 P3 新表）|
| `start.bat` / `stop.bat` | - | 启停脚本 |

---

## 二、本次升级新增内容（1.0.0 → 1.0.4）

### 后端新增模块（8 个 Controller）
- `/api/enterprises` 服务企业档案
- `/api/logistics` 物流服务
- `/api/tax-refunds` 退税业务
- `/api/settlements` 结算收汇
- `/api/insurances` 信保服务
- `/api/financings` 融资协助
- `/api/stats/certification-dashboard` 认定驾驶舱
- `/api/logs` 操作日志查询

### 安全加固
- 登录失败 5 次锁定 30 分钟
- 密码复杂度强制（8-32 位 + 大小写 + 数字 + 特殊字符）
- 操作日志自动记录（AOP）

### 数据库新增（**重要：必须执行迁移**）
- 7 张新表：`biz_service_enterprise` `biz_logistics` `biz_tax_refund` `biz_settlement` `biz_insurance` `biz_financing` `sys_operation_log`
- 3 张表加字段：
  - `biz_contract` 加 `enterprise_id`, `contract_type`
  - `biz_order` 加 `enterprise_id`
  - `biz_customs_declaration` 加 `contract_id`, `order_id`, `enterprise_id`

## 六、常见问题

### Q1: 启动后访问 502 Bad Gateway
- 原因：后端没起来或者 8081 端口没监听
- 排查：`tasklist | findstr java` 看进程；`type logs\app.log` 看末尾
- 常见错因：数据库连接失败（密码错）/ JDK 版本错（必须 17）/ 端口被占

### Q2: 登录提示"登录失败次数过多"
- 原因：5 次密码错误触发 P1 锁定（30 分钟）
- 解决：等 30 分钟，或重启后端清空内存锁定状态

### Q3: 修改密码失败"密码需 8-32 位..."
- 原因：P1 加固后强制密码复杂度
- 要求：8-32 位 + 大写 + 小写 + 数字 + 特殊字符 `!@#$%^&*()_+-=`
- 例如：`Admin@123`

### Q4: 看不到 P3 新模块菜单
- 原因：前端 dist 没替换或浏览器缓存
- 解决：Ctrl+F5 强刷；确认 dist 文件确实是新版（看 index.html 内引用的 hash 文件名是否变了）

### Q5: 报表导出 401 / 403
- 原因：JWT token 过期，重新登录即可

## 七、Java 17 环境

**重要**：1.0.4 用 Java 17 编译，不能用 Java 8 跑。

确认服务器有 JDK 17：

```cmd
java -version
```

期望输出包含 `17.x` 字样。如果还是 1.8：
1. 安装 JDK 17（如 D:\soft\java\jdk-17.0.2）
2. 修改 `start.bat` 用绝对路径调用：
   ```bat
   start /b "D:\soft\java\jdk-17.0.2\bin\java.exe" -jar trade-platform-1.0.4.jar ...
   ```
