## 二、环境准备（首次部署必做）

### 2.1 安装 JDK 17

**重要**：必须 JDK 17，不能用 JDK 8（jar 跑不起来）。

```cmd
:: 下载安装 JDK 17
:: https://www.oracle.com/java/technologies/downloads/#java17
:: 或 https://www.azul.com/downloads/?package=jdk#zulu

:: 验证
java -version
```

期望输出包含 `17.x.x` 字样。

如果服务器同时有多个 Java 版本，**强烈建议**在 `start.bat` 里用绝对路径调用，避免环境变量 PATH 错乱：
```bat
"D:\soft\java\jdk-17.0.2\bin\java.exe" -jar trade-platform-1.0.4.jar ...
```

### 2.2 安装 MySQL 8.0

**推荐方式**：用 phpStudy 集成包（最省事，附带 Nginx）

1. 下载 phpStudy Pro：https://www.xp.cn/
2. 安装到 D:\phpstudy_pro
3. 启动 MySQL 8.0 服务（在 phpStudy 面板）
4. 记下 root 密码（默认 `root`，建议修改）

**或独立安装**：
- https://dev.mysql.com/downloads/mysql/

### 2.3 安装 Nginx

如果用了 phpStudy，已自带 Nginx；否则单独下：
- http://nginx.org/en/download.html

## 四、全新部署（7 步）

> 适用：服务器上从未部署过本系统

### 第 1 步：拷贝部署包到服务器

```cmd
:: 假设解压到 D:\deploy\autoWorkPlatform
mkdir D:\deploy\autoWorkPlatform
:: 把 deploy/ 整个目录内容复制过去
```

### 第 2 步：初始化数据库

```cmd
:: 用 phpStudy 自带的 mysql.exe
"D:\phpstudy_pro\Extensions\MySQL8.0.12\bin\mysql.exe" -uroot -p<你的密码>

:: 在 MySQL 命令行执行
mysql> CREATE DATABASE trade_platform DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;
mysql> exit

:: 导入完整表结构 + 初始数据（含 admin 账号 + 字典配置）
"D:\phpstudy_pro\Extensions\MySQL8.0.12\bin\mysql.exe" -uroot -p<你的密码> trade_platform < D:\deploy\autoWorkPlatform\sql\init.sql
```

> ⚠️ 这一步会创建所有表，并写入 admin 用户（密码 `Admin@123`）和基础字典数据。

### 第 3 步：修改生产配置

编辑 `D:\deploy\autoWorkPlatform\application-prod.yml`，**改 3 处**：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/trade_platform?...   # 数据库地址（一般不改）
    username: root
    password: YOUR_PASSWORD          # ← 改成你的 MySQL root 密码

jwt:
  secret: YOUR_JWT_SECRET_CHANGE_THIS_32_CHARS   # ← 改成 32+ 位随机字符串
                                                  # 例如：8x7Hkn2vQp9mAcR3bSjF6tYxZw5lEdGu

file:
  upload-path: D:/deploy/autoWorkPlatform/uploads/   # ← 改成你的实际路径
```

> 💡 **JWT secret 生成**：在 PowerShell 里跑：
> ```powershell
> -join ((48..57) + (65..90) + (97..122) | Get-Random -Count 40 | % {[char]$_})
> ```

### 第 4 步：配置 Nginx

把 `dist/` 目录拷到 Nginx 的静态目录（默认 `<nginx>/html/dist`）：

```cmd
:: 假设 nginx 在 D:\phpstudy_pro\Extensions\Nginx1.25.3
xcopy /E /I D:\deploy\autoWorkPlatform\dist D:\phpstudy_pro\Extensions\Nginx1.25.3\html\dist
```

编辑 Nginx 配置（如 `D:\phpstudy_pro\Extensions\Nginx1.25.3\conf\nginx.conf`），把下面这段塞到 `http { ... }` 内：

```nginx
server {
    listen       80;
    server_name  localhost;
    client_max_body_size 50m;       # 允许上传大文件

    # 前端静态文件
    location / {
        root   html/dist;
        index  index.html;
        try_files $uri $uri/ /index.html;     # SPA 路由必备
    }

    # 后端 API 反向代理
    location /api {
        proxy_pass         http://127.0.0.1:8081;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
    }

    # 上传文件直接访问（绕过后端）
    location /uploads {
        alias D:/deploy/autoWorkPlatform/uploads;
        autoindex off;
    }
}
```

启动 / 重载 Nginx：

```cmd
:: 启动（首次）
cd D:\phpstudy_pro\Extensions\Nginx1.25.3
nginx.exe

:: 重载（修改配置后）
nginx.exe -s reload

:: 停止
nginx.exe -s stop
```

### 第 5 步：启动后端

```cmd
cd D:\deploy\autoWorkPlatform
start.bat
```

正常输出：

```
Starting backend...
[OK] Backend started on port 8081
```

如果失败，看 `logs\app.log` 末尾。

### 第 6 步：验证

```cmd
:: 验证后端
curl http://127.0.0.1:8081/api/auth/login -X POST -H "Content-Type: application/json" -d "{\"tenantCode\":\"GUOHE\",\"username\":\"admin\",\"password\":\"Admin@123\"}"
```

期望返回 `{"code":200, "data":{"token":"..."}}`

浏览器访问 `http://服务器IP/`，用以下账号登录：

```
企业号：GUOHE
用户名：admin
密码：Admin@123
```

### 第 7 步：首次登录后必做

1. **立即修改 admin 密码**（个人中心 → 修改密码）
   - 新密码要求：8-32 位 + 大小写字母 + 数字 + 特殊字符
2. **创建租户**（租户管理 → 添加租户）
3. **审批新用户注册**（用户管理 → 用户审批）

## 八、常见问题

### Q1：访问页面报 502 Bad Gateway

**原因**：Nginx 收到请求但后端没起来。

**排查**：
```cmd
:: 1. 看 8081 有没有监听
netstat -ano | findstr ":8081"

:: 2. 如果没有，看 java 进程
tasklist | findstr java

:: 3. 看后端日志
type D:\deploy\autoWorkPlatform\logs\app.log
```

**常见错因**：
- 数据库连接失败（密码错 / MySQL 没启动）
- JDK 版本错（必须 17）
- 端口 8081 被占
- application-prod.yml 路径错（要用 `--spring.config.location=` 指定）

### Q2：登录提示"系统繁忙，请稍后再试"

**原因**：通常是数据库表不全，看 `logs\app.log` 找 `SQLSyntaxErrorException` 关键字。

**解决**：重跑 `sql/init.sql`（或对应模块的迁移 SQL）。

### Q3：登录提示"登录失败次数过多"

**原因**：触发了 P1 安全锁定（5 次失败 → 锁 30 分钟）。

**解决**：
- 等 30 分钟自动解锁
- 或重启后端清空内存锁定状态

### Q4：修改密码失败"密码需 8-32 位..."

**P1 强制密码复杂度**：
- 长度 8-32 位
- 必须含大写字母
- 必须含小写字母
- 必须含数字
- 必须含特殊字符 `!@#$%^&*()_+-=`

**合规示例**：`Admin@123`、`Trade2026!`、`MyP@ssw0rd`

### Q5：上传附件失败

**排查顺序**：

1. 文件大小是否超限（默认后端 20MB，nginx 50MB）
2. `uploads/` 目录是否存在且可写
3. `application-prod.yml` 里的 `file.upload-path` 路径是否正确
4. 看 `logs\app.log` 末尾

### Q6：前端能打开但 API 都报 404

**原因**：Nginx 没配 `/api` 反向代理。

**解决**：参考第 4 步 Nginx 配置，加上 `location /api { proxy_pass ... }`。

### Q7：登录后右上角企业名空白

**原因**：JWT secret 在 application-prod.yml 中没改（或被改回默认值），导致 token 失效。

**解决**：改 JWT secret，重启后端，重新登录。

## 九、性能调优（可选）

### 后端 JVM 内存

默认堆 256MB，数据量大时建议加：

修改 `start.bat`：
```bat
start /b java -Xms512m -Xmx1024m -jar trade-platform-1.0.4.jar --spring.config.location=application-prod.yml > logs\app.log 2>&1
```

### MySQL 优化

`my.ini`：
```ini
innodb_buffer_pool_size = 1G        # 内存的 50-70%
max_connections = 200
slow_query_log = 1
```

### Nginx gzip

`nginx.conf` 的 `http { ... }` 内加：
```nginx
gzip on;
gzip_min_length 1k;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
gzip_vary on;
```
